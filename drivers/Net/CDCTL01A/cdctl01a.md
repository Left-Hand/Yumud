# CDCTL01A 芯片驱动开发业务手册

## 一、芯片核心定位与驱动开发基础

### 1.1 芯片核心应用场景

CDCTL01A 是一款专门适配 CDBUS 通信协议的控制芯片，主要用于实现设备间的可靠数据传输，核心支撑四大业务场景：



* 时钟系统管理（选择高频 / 低频时钟，适配不同通信速率与功耗需求）；

* 通信模式定制（根据总线拓扑与传输需求，配置收发模式与数据处理规则）；

* 引脚与中断控制（确保芯片与主机、总线的电气特性匹配，实现中断信号正常交互）；

* 数据收发辅助（优化数据传输的同步性与完整性，降低通信错误率）。

### 1.2 驱动开发核心前提



* 需获取芯片手册中寄存器的实际物理地址（用于驱动中硬件读写操作）；

* 需实现 “读取寄存器状态”“写入配置参数” 的基础硬件交互能力；

* 所有配置需遵循 “配置后生效” 规则，部分关键操作（如时钟切换、芯片复位）需按手册规定的时序执行，避免功能异常。

## 二、核心寄存器业务功能与配置说明

以下按 “业务模块” 分类，清晰说明每个寄存器的作用、地址、各配置项含义及使用注意事项，配置项按寄存器 “最低位到最高位” 的顺序描述（对应手册中 Bit0 至 Bit7 的字段定义）。

### 2.1 芯片版本校验模块

#### 寄存器名称：硬件版本寄存器（地址：0x00）



* 核心作用：驱动启动时确认芯片型号与版本兼容性，避免硬件不匹配导致异常；

* 配置说明：只读，整字节为版本号（编码规则参考手册版本定义表）；

* 驱动使用：初始化读取并与支持版本比对，不匹配则终止并提示。

### 2.2 时钟系统管理模块

#### 2.2.1 时钟控制寄存器（地址：0x01）



* 核心作用：选择系统时钟源，触发软件复位（异常恢复 / 初始化重置）；

* 配置项说明（从最低位到最高位）：

1. 第 1 位：时钟源选择 \[1=PLL 时钟（高速通信）/0 = 晶振时钟（低功耗）]

2. 第 2-7 位：预留位，配置为 0

3. 第 8 位：软件复位 \[1 = 触发复位（完成后自动清 0）/0 = 无复位]

* 业务逻辑：复位需在时钟源选择前执行；切换时钟源前需确认目标时钟就绪（通过时钟状态寄存器）。

#### 2.2.2 PLL 控制寄存器（地址：0x04）



* 核心作用：控制 PLL 时钟启动、关闭与节能，配合时钟控制寄存器实现高频时钟管理；

* 配置项说明（从最低位到最高位）：

1. 第 1 位：PLL 节能模式 \[1 = 睡眠（停时钟，低功耗）/0 = 正常工作（稳定输出）]

2. 第 2-4 位：预留位，配置为 0

3. 第 5 位：PLL 启动 \[1 = 启动 PLL/0 = 关闭 PLL]

4. 第 6-8 位：预留位，配置为 0

* 业务逻辑：仅时钟源选择 PLL 时生效；启动后需等待时钟稳定（通过时钟状态寄存器）再配置后续参数。

#### 2.2.3 时钟状态寄存器（地址：0x07）



* 核心作用：反馈时钟系统工作状态，用于状态判断与容错（只读）；

* 状态项说明（从最低位到最高位）：

1. 第 1 位：当前时钟源为晶振 \[1 = 是 / 0 = 否]

2. 第 2 位：当前时钟源为 PLL \[1 = 是 / 0 = 否]

3. 第 3 位：时钟源切换中 \[1 = 切换中（禁止配置时钟参数）/0 = 切换完成]

4. 第 4-8 位：预留位，仅读取

* 驱动使用：时钟切换后，轮询至 “切换中” 为 0 且目标时钟源为 1，再配置波特率 / 通信模式。

#### 2.2.4 空闲等待长度寄存器（地址：0x02）



* 核心作用：配置总线空闲状态的等待时长，适配不同总线响应速度；

* 配置项说明：整字节为等待长度（单位：时钟周期，取值范围参考手册）；

* 操作属性：读写；

* 配置注意：长度过短易导致数据丢失，过长降低传输效率，需按总线特性适配。

#### 2.2.5 发送允许长度低位寄存器（地址：0x05）

#### 2.2.6 发送允许长度高位寄存器（地址：0x06）



* 核心作用：联合配置最大发送允许时长，避免总线占用超时；

* 配置项说明：两寄存器组成 16 位数值（低位 + 高位），单位：时钟周期；

* 操作属性：读写；

* 业务逻辑：需与时钟源频率匹配，高频时钟时需对应增大数值以保证发送完整性。

#### 2.2.7 最大空闲长度低位寄存器（地址：0x08）

#### 2.2.8 最大空闲长度高位寄存器（地址：0x09）



* 核心作用：配置总线最大空闲时长，超过则触发链路重置；

* 配置项说明：两寄存器组成 16 位数值（低位 + 高位），单位：时钟周期；

* 操作属性：读写；

* 配置注意：需根据总线通信周期调整，避免误触发重置或链路超时。

### 2.3 通信模式配置模块

#### 寄存器名称：功能配置寄存器（地址：0x03）



* 核心作用：配置 CDBUS 通信核心规则（引脚模式、数据处理、通信模式）；

* 配置项说明（从最低位到最高位）：

1. 第 1 位：TOE 引脚输出模式 \[1 = 推挽（单设备，强驱动）/0 = 开漏（多设备，总线拓扑）]

2. 第 2 位：TO 引脚电平反转 \[1 = 反转（适配总线电平）/0 = 正常电平]

3. 第 3 位：CRC 校验处理 \[1 = 软件处理（驱动实现）/0 = 硬件处理（芯片自动）]

4. 第 4 位：错误帧保存 \[1 = 保存（调试分析）/0 = 丢弃（正常通信）]

5. 第 5 位：CDBUS A 模式 \[1 = 启用 / 0 = 禁用]

6. 第 6 位：CDBUS B 模式 \[1 = 启用 / 0 = 禁用]

7. 第 7 位：全双工模式 \[1 = 启用（双向同时收发）/0 = 半双工（单向交替）]

8. 第 8 位：预留位，配置为 0

* 业务逻辑：A/B 模式不可同时启用；全双工模式需同步配置引脚参数避免冲突。

### 2.4 数据收发辅助模块

#### 寄存器名称：发送前导码长度寄存器（地址：0x0A）



* 核心作用：配置发送数据包前导码长度，辅助接收端时钟同步；

* 配置说明：整字节为长度（单位：比特，取值范围参考手册）；

* 操作属性：读写；

* 配置注意：过短导致同步失败，过长占用带宽；高速通信可缩短，低速可延长。

#### 2.4.1 地址过滤寄存器（地址：0x0B）



* 核心作用：配置地址过滤规则，实现目标地址匹配；

* 配置项说明（从最低位到最高位）：

1. 第 1-4 位：过滤模式选择（编码规则参考手册：如精确匹配 / 掩码匹配）

2. 第 5-8 位：地址匹配使能 \[1 = 启用地址过滤 / 0 = 禁用（接收所有数据）]

* 操作属性：读写；

* 业务逻辑：启用后仅接收匹配目标地址的数据，减少无效数据占用资源。

#### 2.4.2 接收地址寄存器（地址：0x19）



* 核心作用：存储目标接收地址，配合地址过滤寄存器实现匹配；

* 配置说明：整字节为目标地址（8 位地址编码，需与总线设备地址一致）；

* 操作属性：读写；

* 配置注意：需与地址过滤规则对应，否则过滤功能失效。

#### 2.4.3 过滤掩码 1 寄存器（地址：0x18）

#### 2.4.4 过滤掩码 2 寄存器（地址：0x1B）



* 核心作用：配置地址过滤掩码，灵活调整地址匹配范围；

* 配置说明：整字节为掩码值（与接收地址按位与运算后判断匹配）；

* 操作属性：读写；

* 业务逻辑：掩码位为 1 时对应地址位必须匹配，为 0 时忽略该位，需与过滤模式配合使用。

### 2.5 波特率配置模块

#### 2.5.1 低速分频系数低位寄存器（地址：0x0C）

#### 2.5.2 低速分频系数高位寄存器（地址：0x0D）

#### 2.5.3 高速分频系数低位寄存器（地址：0x0E）

#### 2.5.4 高速分频系数高位寄存器（地址：0x0F）



* 核心作用：配置通信波特率，分为低速 / 高速模式，通过分频系数调整速率；

* 配置项说明：每组寄存器（低位 + 高位）组成 16 位分频系数，速率计算公式：波特率 = 时钟频率 /(分频系数 + 1)；

* 操作属性：读写；

* 配置注意：需根据选择的时钟源频率（晶振 / PLL）计算分频值，确保波特率误差在允许范围。

### 2.6 中断管理模块

#### 2.6.1 中断标志寄存器（地址：0x10）



* 核心作用：反馈中断触发类型（只读，清除方式参考手册）；

* 状态项说明（从最低位到最高位）：

1. 第 1 位：接收完成中断 \[1 = 触发 / 0 = 未触发]

2. 第 2 位：发送完成中断 \[1 = 触发 / 0 = 未触发]

3. 第 3 位：接收错误中断 \[1 = 触发（帧错误 / CRC 错误）/0 = 未触发]

4. 第 4 位：总线空闲中断 \[1 = 触发 / 0 = 未触发]

5. 第 5-8 位：预留中断位，仅读取

* 驱动使用：中断响应时读取，判断中断类型并执行对应处理逻辑。

#### 2.6.2 中断掩码寄存器（地址：0x14）



* 核心作用：控制各中断类型的使能 / 禁用；

* 配置项说明（从最低位到最高位）：

1. 第 1 位：接收完成中断使能 \[1 = 启用 / 0 = 禁用]

2. 第 2 位：发送完成中断使能 \[1 = 启用 / 0 = 禁用]

3. 第 3 位：接收错误中断使能 \[1 = 启用 / 0 = 禁用]

4. 第 4 位：总线空闲中断使能 \[1 = 启用 / 0 = 禁用]

5. 第 5-8 位：预留中断位，配置为 0

* 操作属性：读写；

* 业务逻辑：需在启用中断功能前配置，避免不必要的中断触发。

### 2.7 数据收发控制模块

#### 2.7.1 接收数据寄存器（地址：0x11）



* 核心作用：存储接收到的完整数据帧（只读）；

* 配置说明：整字节为接收数据（需结合接收完成中断或状态位读取）；

* 驱动使用：收到接收完成中断后读取，避免数据覆盖。

#### 2.7.2 发送数据寄存器（地址：0x15）



* 核心作用：存储待发送的数据帧（只写）；

* 配置说明：写入数据后触发发送（需确保发送使能）；

* 驱动使用：发送前需检查发送就绪状态，避免数据丢失。

#### 2.7.3 接收控制寄存器（地址：0x16）



* 核心作用：配置接收相关控制参数；

* 配置项说明（从最低位到最高位）：

1. 第 1 位：接收使能 \[1 = 启用接收 / 0 = 禁用接收]

2. 第 2 位：接收溢出中断使能 \[1 = 启用 / 0 = 禁用]

3. 第 3-8 位：预留位，配置为 0

* 操作属性：读写；

* 业务逻辑：禁用接收时，芯片将丢弃所有接收数据。

#### 2.7.4 发送控制寄存器（地址：0x17）



* 核心作用：配置发送相关控制参数；

* 配置项说明（从最低位到最高位）：

1. 第 1 位：发送使能 \[1 = 启用发送 / 0 = 禁用发送]

2. 第 2 位：发送溢出中断使能 \[1 = 启用 / 0 = 禁用]

3. 第 3-8 位：预留位，配置为 0

* 操作属性：读写；

* 业务逻辑：禁用发送时，写入发送数据寄存器的数据不会触发发送。

### 2.8 引脚与中断控制模块

#### 2.8.1 中断引脚控制寄存器（地址：0x05）



* 核心作用：配置中断输出引脚电气特性，确保主机正确识别中断信号；

* 配置项说明（从最低位到最高位）：

1. 第 1-4 位：预留位，配置为 0

2. 第 5 位：中断引脚输出模式 \[1 = 推挽（短距离，强驱动）/0 = 开漏（多设备共享，需外部上拉）]

3. 第 6-8 位：预留位，配置为 0

* 业务逻辑：启用中断前完成配置，避免中断信号传输异常。

#### 2.8.2 RE 引脚控制寄存器（地址：0x06）



* 核心作用：控制接收使能（RE）引脚状态与模式，决定接收功能是否启用；

* 配置项说明（从最低位到最高位）：

1. 第 1 位：RE 引脚电平 \[1 = 高电平（通常接收启用）/0 = 低电平（通常接收禁用），以手册为准]

2. 第 2-3 位：预留位，配置为 0

3. 第 4 位：RE 引脚输出模式 \[1 = 推挽（直接控制接收电路）/0 = 开漏（需外部电阻，适配总线）]

4. 第 5-8 位：预留位，配置为 0

* 驱动使用：接收前确保配置为启用状态；切换收发模式时同步调整电平。

## 三、核心业务流程与配置规范

### 3.1 芯片初始化标准流程（驱动必须遵循）



1. 读取硬件版本寄存器（0x00），校验版本兼容性；

2. 配置时钟控制寄存器（0x01），触发软件复位，延时等待完成；

3. 若使用 PLL 时钟：配置 PLL 控制寄存器（0x04）启动 PLL，通过时钟状态寄存器（0x07）确认稳定；

4. 配置时钟控制寄存器（0x01），选择目标时钟源；

5. 配置波特率分频寄存器（0x0C-0x0F），设置通信速率；

6. 配置功能配置寄存器（0x03），确定通信模式、CRC 处理等参数；

7. 配置地址过滤与接收地址寄存器（0x0B、0x19、0x18、0x1B），设置地址匹配规则；

8. 配置中断引脚与 RE 引脚控制寄存器（0x05、0x06），匹配电气特性；

9. 配置中断掩码寄存器（0x14），启用所需中断类型；

10. 配置发送前导码长度寄存器（0x0A）及收发控制寄存器（0x16、0x17），启用收发功能；

11. 完成初始化，进入数据收发就绪状态。

### 3.2 关键业务场景操作规范



| 业务场景   | 操作步骤                                                     | 注意事项            |
| ------ | -------------------------------------------------------- | --------------- |
| 时钟源切换  | 1. 禁用收发功能；2. 配置目标时钟源控制寄存器；3. 轮询时钟状态寄存器；4. 确认切换完成后重新配置波特率 | 切换中禁止配置时钟相关参数   |
| 软件复位   | 1. 可选保存当前配置；2. 配置时钟控制寄存器复位位；3. 延时等待；4. 重新执行完整初始化流程       | 复位后寄存器恢复默认值，需重配 |
| 通信模式变更 | 1. 停止收发；2. 修改功能配置寄存器；3. 重新配置引脚与前导码长度；4. 重启收发功能           | 变更后需测试通信稳定性     |
| 错误处理   | 1. 读取时钟状态寄存器判断时钟异常；2. 读取中断标志寄存器确认错误类型；3. 针对性恢复（时钟 / 链路）  | 错误处理后校验通信参数     |

## 四、补充说明



* 寄存器地址基于 CDCTL01A 手册常规分配，实际开发需以手册标注的物理地址为准；

* 部分寄存器（如中断标志清除、PLL 稳定延时）的细节操作需参考手册具体时序要求；

* 未明确取值范围的配置项（如长度、分频系数），需按手册给出的范围配置，避免超出芯片支持能力。
