# Datasheet 寄存器提取与代码生成需求规范

## 一、核心目标

基于提供的 datasheet，提取寄存器相关章节，生成 **规范 markdown 文档** 和 **工程化可用的 C++ 寄存器结构体代码**。

## 二、Markdown 文档输出要求

### 2.1 结构规范



1. 按 datasheet 原有章节层级组织（如 “全局寄存器”“设备模式寄存器”“主机模式寄存器”），层级使用 `##` `###` `####` 区分；

2. 每个寄存器模块先列出 **寄存器列表**（含名称、访问地址、描述、复位值），再逐一详细说明单个寄存器；

3. 单个寄存器需包含：位号、位名称、访问权限（RW/RO）、描述、复位值，用表格呈现；

4. 关键注释（如保留位说明、模式复用说明、特殊操作要求）需完整保留并标注。

### 2.2 格式要求



* 寄存器名称、位名称使用 **加粗** 突出（如 `R8_USB_CTRL`、`RB_UC_HOST_MODE`）；

* 代码相关内容（如地址、复位值）使用 `code` 格式（如 `0x40023400`、`0x06`）；

* 表格列对齐，字段清晰（名称、访问地址、描述、复位值 / 位、名称、访问、描述、复位值）。

## 三、C++ 代码生成要求

### 3.1 命名规则



1. 寄存器结构体统一命名为 `R32_xxxxx`（如 `R32_USB_CTRL`、`R32_UEP_CONFIG`），遵循 “寄存器原始名称前缀 + 功能描述”；

2. 结构体成员（位段）名称与 datasheet 完全一致（如 `RB_UC_HOST_MODE`、`MASK_UEP_T_TOG`）；

3. 功能方法命名：读取方法用 `get_xxx()`，设置方法用 `set_xxx()`，使能 / 禁止方法用 `enable_xxx()`/`disable_xxx()`，检查方法用 `is_xxx()`。

### 3.2 结构体规范


1. 位段定义精准匹配 datasheet（位宽、位顺序完全一致），保留位明确标注 `__RESV__` 或 `RESERVED`；

2. 每个结构体添加 **详细注释**：寄存器功能、偏移地址、复位值、关键说明；

3. 结构体大小校验：使用宏 `CHECK_R32(T)`（32 位寄存器）、`CHECK_R8(T)`（8 位寄存器，兼容 32 位访问），以检查寄存器的大小正确且位域填充无误


### 3.3 方法实现要求

1. 禁止修改 datasheet 原始位名称、位宽、功能描述；

2. 禁止使用 `const_cast`、 `reinterpret_cast` 等不安全转换；

3. 所有方法使用 `constexpr` 修饰（如 `constexpr bool is_host_mode() const`）；

4. 方法功能明确：覆盖寄存器核心操作（设置位值、使能 / 禁止功能、读取状态、清零标志等），参数添加合法性校验（如地址对齐、端点号范围）。

5. 禁止遗漏保留位、复位值、关键操作注释（如 DMA 地址 4 字节对齐要求）。

6. 注释规范遵守doxygen规范。

### 3.4 额外规范

1. 提供寄存器基地址与映射定义（如 `static constexpr USBHS_BASE = 0x40023400;`），支持工程直接使用；

2. 所有有返回值的函数/结构体标注为 `[[nodiscard]]`

3. 适配 C++20 标准，使用 `std::bit_cast` 进行类型转换（替代 `reinterpret_cast`），避免未定义行为；

4. 枚举类型：响应类型、触发位类型等定义枚举（如 `TxResp`、`RxTog`），提高代码可读性。
