## 阶段 1：前向转换验证 (FP32 → FP8)

### 目标
验证 `fp32_to_fp8_e4m3_nonfpu()` 函数的正确性

### 脚本：verify_fp8_e4m3.py

#### 验证策略

**策略 1：约束满足性（Z3 求解器）**
```python
def verify_fp32_to_fp8_e4m3() -> bool:
    s = Solver()
    
    # 约束 1：溢出条件 (exp > 7)
    s.add(Implies(exp_unbias > 7, result == 0x7E | (sign << 7)))
    
    # 约束 2：下溢条件 (exp < -8)
    s.add(Implies(exp_unbias < -8, result == sign << 7))
    
    # 约束 3：正常范围 (-8 <= exp <= 7)
    s.add(Implies(And(exp_unbias >= -8, exp_unbias <= 7), True))
    
    return s.check() == sat
```

**策略 2：边界值测试**

## 阶段 2：反向转换初步验证 (FP8 → FP32)

### 目标
验证 `fp8_e4m3_to_fp32_nonfpu()` 函数的正确性

### 脚本：verify_fp8_e4m3_reverse.py

#### 验证策略

**分类验证法**：将 FP8-E4M3 的所有情况分类验证

```python
# 情况 1：零值 (exp=0, mant=0)
if exp == 0 and mant == 0:
    f32_bits = 0  # ±0，保留符号

# 情况 2：非规范化数 (exp=0, mant≠0)
elif exp == 0 and mant != 0:
    # 值 = 2^-6 * (mant/8)
    exp32 = 121
    mant32 = mant << 17

# 情况 3：规范化数 (exp ∈ [1, 14])
elif 1 <= exp <= 14:
    # 值 = 2^(exp-7) * (1 + mant/8)
    exp32 = exp + 120
    mant32 = (8 + mant) << 20

# 情况 4：特殊值 (exp=15)
elif exp == 0xF:
    # Inf 或 NaN
    f32_bits = (0xFF << 23) | (mant << 20)
```

## 阶段 3：Bug 诊断 (FP8 → FP32)

### 目标
找出反向转换函数的确切 bug

### 脚本：verify_fp8_detailed.py

#### 诊断方法

对比原始实现和修正实现逐值转换

```python
def test_value(original_fp32, desc):
    # 步骤 1：FP32 → FP8
    fp8 = fp32_to_fp8_e4m3(original_fp32)
    
    # 步骤 2：FP8 → FP32（原始实现）
    fp32_original = fp8_to_fp32_original(fp8)
    
    # 步骤 3：FP8 → FP32（修正实现）
    fp32_fixed = fp8_to_fp32_fixed(fp8)
    
    # 步骤 4：对比
    print(f"原始: {fp32_original} {'✓' if fp32_original == original_fp32 else '✗ ERROR'}")
    print(f"修正: {fp32_fixed}   {'✓' if fp32_fixed == original_fp32 else '✗'}")
```

#### 诊断结果

```
1.0 (exp=127, mant=0)
  原始代码→: 1.0 (0x3F800000) ✓
  修正代码→: 1.0 (0x3F800000) ✓

0.5 (exp=126, mant=0)
  原始代码→: 1.0 (0x3F800000) ✗ ERROR   ← 翻倍了！
  修正代码→: 0.5 (0x3F000000) ✓

2.0 (exp=128, mant=0)
  原始代码→: 4.0 (0x40800000) ✗ ERROR   ← 翻倍了！
  修正代码→: 2.0 (0x40000000) ✓

8.0 (exp=130, mant=0)
  原始代码→: 16.0 (0x41800000) ✗ ERROR  ← 翻倍了！
  修正代码→: 8.0 (0x41000000) ✓

0.001953125 (非规范化数)
  原始代码→: 0.0 (0x00000000) ✗ ERROR
  修正代码→: 0.001953125 expected...
```

### 问题诊断

#### Bug 1：额外的隐藏位设置 ❌

**原始代码**
```cpp
uint32_t f32_exp = exp + (127 - 7);  // exp + 120
uint32_t f32_mant = (uint32_t)mant << 20;
f32_bits = (f32_exp << 23) | f32_mant | (1U << 23);  // ← BUG!
//                                      ^^^^^^^^^^^
//                        IEEE 754 隐藏位不应显式设置
```

**分析**

- `(1 << 23)` = `0x800000` 在尾数字段中添加了额外的 bit
- IEEE 754 格式中，隐藏位（hidden bit）是隐含的，值总是 1.0
- 显式设置会导致数值变化
- 正确做法：不设置，隐藏位由格式隐含

**位表示对比**

假设 FP8 值对应 FP32(0.5)：

```
原始代码：
  f32_exp = 6 + 120 = 126
  f32_mant = 0 << 20 = 0
  f32_bits = (126 << 23) | 0 | (1<<23)
           = 0x3F000000 | 0x800000
           = 0x3F800000  ← 0.5 + 0x800000 = 1.0 ✗

修正代码：
  f32_exp = 6 + 120 = 126
  f32_mant = 0 << 20 = 0
  f32_bits = (126 << 23) | 0
           = 0x3F000000  ← 0.5 ✓
```

#### Bug 2：非规范化数未处理 ❌

**原始代码**
```cpp
} else {
    // 注释说处理 exp ∈ [1, 14]
    // 但实际上 exp=0 && mant≠0 也进入这里！
    uint32_t f32_exp = exp + (127 - 7);  // exp=0 时，变成 120 ✗
    ...
}
```

**分析**

- 非规范化数（subnormal）有特殊的指数规则
- FP8 中：exp=0, mant≠0 → 值 = 2^-6 * (mant/8)
- 对应 FP32：固定 exp=121（而非 exp+120）
- 原始代码没有区分，导致转换错误

**对比**

```
FP8 非规范化数（exp=0, mant=1）：
  值 = 2^-6 * (1/8) = 2^-9

原始代码：
  f32_exp = 0 + 120 = 120
  f32_bits 对应 2^(-7)  ✗ 错误

修正代码：
  f32_exp = 121
  f32_bits 对应 2^(-6) * (1/8) = 2^-9  ✓
```

### 结论
❌ **发现 2 个关键 bug，需要修复**

## 阶段 4：修复与验证合并

### 修复方案

#### 文件 1：core/math/float/fp8.hpp

**修复 `fp8_e4m3_to_fp32_nonfpu()` 函数**

```cpp
static constexpr float fp8_e4m3_to_fp32_nonfpu(uint8_t value){
    uint8_t sign = (value >> 7) & 0x1;
    uint8_t exp = (value >> 3) & 0xF;
    uint8_t mant = value & 0x7;

    uint32_t f32_bits = 0;
    
    if (exp == 0 && mant == 0) {
        // 零值：±0，保留符号
        f32_bits = 0;
    } else if (exp == 0xF) {
        // 特殊值：Inf 或 NaN
        f32_bits = (0xFF << 23) | (mant << 20);
    } else if (exp == 0 && mant != 0) {
        // 非规范化数：值 = 2^-6 * (mant/8)
        // FP32 表示：exp32 = 121，mant32 = mant << 20
        uint32_t f32_exp = 121;
        uint32_t f32_mant = (uint32_t)mant << 20;
        f32_bits = (f32_exp << 23) | f32_mant;
    } else {
        // 规范化数：exp ∈ [1, 14]
        // 值 = 2^(exp-7) * (1 + mant/8)
        // FP32 表示：exp32 = exp + 120，mant32 = mant << 20
        uint32_t f32_exp = exp + 120;
        uint32_t f32_mant = (uint32_t)mant << 20;
        f32_bits = (f32_exp << 23) | f32_mant;
        // 移除了 (1U << 23) ✓
    }
    
    f32_bits |= (uint32_t)sign << 31;
    return std::bit_cast<float>(f32_bits);
}
```

**关键变更**
- ✓ 移除了 `(1U << 23)`
- ✓ 添加了非规范化数的单独处理（exp=121）
- ✓ 明确的 unreachable 约束
